# Codex Playbook

## Default behavior

- inspect the repository before changing code
- prefer existing patterns
- keep edits scoped
- update documentation when structure changes

## Required outputs

When making structural changes, also update:

- `docs/project-context.md` if the big picture changes
- an ADR if the architecture changes
- a runbook if operations change

## First-class concern

Repository continuity across accounts matters as much as code correctness.
